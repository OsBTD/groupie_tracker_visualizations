Welcome to Groupie Tracker

Explore artist information, concert dates, and locations through a clean interface that aggregates data from multiple APIs. Discover band histories and tour details in one centralized platform.

Description

This project, developed as part of our web development curriculum, focuses on API integration and data visualization. Our primary goals were to create an intuitive user interface, implement robust error handling, and ensure seamless data synchronization between multiple data sources.

Technologies

  • Go (Golang)
  • HTML5
  • CSS3
  • JavaScript
  • REST API Integration

Installation

To run locally:

1. Clone the repository:
git clone https://learn.zone01oujda.ma/git/oatmani/groupie-tracker

2. Navigate to project directory:
cd groupie-tracker

3. Run the Go server:
go run main.go

4. Open in browser:
http://localhost:8080

Docker Setup

1. Build image:
docker build -t groupie-tracker .

2. Run container:
docker run -p 8080:8080 groupie-tracker

Usage

  • Artist Exploration: Browse artists through interactive cards
  • Detail View: Click cards to see members, albums, and tour dates
  • Concert Tracking: View locations and dates for past/future events
  • Responsive Design: Works seamlessly across desktop and mobile

Implementation Details

API Integration

Combines data from multiple endpoints while handling rate limiting and errors

Data Processing

Go-based backend with concurrent fetching and data normalization

Error Handling

Comprehensive error pages with proper HTTP status codes

Client-Side Routing

Hash-based navigation for artist details without page reloads
